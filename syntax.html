4. Синтаксис

Язык регистрозависимый, все ключевые слова пишутся в нижнем регистре. Следующие зарезервированные слова не могут быть использованы для имён:

  • and Это логический оператор «И», который возвращает первый операнд, если он false или nil, иначе возвращает второй операнд. Он реализует «короткое замыкание»: если первый элемент ложен, второй не вычисляется. Часто используется для проверки условий и выбора значений.
    Подробнее →
  • end Это зарезервированное ключевое слово, обозначающее закрытие структурного блока кода, такого как функции (function), условные операторы (if/else), циклы (for, while, repeat) или блоки do. Оно необходимо для завершения управляющих конструкций.
    Подробнее →
  • if Это базовый управляющий условный оператор, используемый для принятия решений в коде. Он проверяет условие и, если оно истинно (true), выполняет соответствующий блок кода. Конструкция позволяет создавать интерактивные сценарии, используя if, then, elseif, else и завершается обязательным ключевым словом end.
    Подробнее →
  • or Это логическое ИЛИ, которое возвращает первый аргумент, если он не false или nil, иначе возвращает второй. В отличие от многих языков, он возвращает само значение, а не булево, и поддерживает «короткое замыкание» (не вычисляет второй операнд, если первый истинен).
    Подробнее →
  • until В Lua — это ключевое слово, завершающее цикл repeat...until. Оно повторяет блок кода, пока указанное условие не станет истинным (true). В отличие от while, цикл repeat гарантированно выполняется хотя бы один раз.
    Подробнее →
  • break Это оператор управления потоком, используемый для немедленного завершения ближайшего цикла (while, repeat или for). После break выполнение программы продолжается с первой команды, следующей за циклом.
    Подробнее →
  • false Это один из двух логических типов данных (boolean), представляющий значение «ложь». В условных операторах false (наряду с nil) считается ложным.
    Подробнее →
  • in Это служебная часть синтаксиса, используемая исключительно в конструкциях циклов for для итерации по значениям, последовательностям или таблицам. Оно связывает переменную цикла с итератором (например, pairs или ipairs).
    Подробнее →
  • repeat Это цикл с постусловием, который гарантированно выполняет блок кода хотя бы один раз, проверяя условие завершения (until) после выполнения тела цикла. Он работает противоположно while.
    Подробнее →
  • while Это управляющий оператор цикла с предусловием, который повторяет блок кода, пока заданное логическое условие истинно (true). Сначала проверяется условие: если оно ложно, тело цикла не выполняется ни разу.
    Подробнее →
  • do Это ключевое слово, обозначающее начало блока выполнения кода в управляющих конструкциях (for, while) или определяющее границы локального блока. Оно означает "сделать" и завершается словом end.
    Подробнее →
  • not Это логический оператор «НЕ», используемый для инверсии булевых значений. Он меняет true на false и наоборот. В Lua nil и false считаются ложными, поэтому not перед ними вернет true.
    Подробнее →
  • then Это ключевое слово, используемое в условных конструкциях if и elseif для обозначения начала блока кода, который должен выполниться, если проверенное условие истинно. Отделяет условие от выполняемых действий.
    Подробнее →
  • else Это ключевое слово, используемое в конструкциях if-then-else-end для выполнения блока кода, если условие if является ложным. Оно обеспечивает ветвление логики.
    Подробнее →
  • for Это управляющая конструкция, создающая цикл для многократного выполнения блока кода. В Lua существует два вида: числовой (итерация по счетчику) и обобщенный (итерация по таблицам/функциям).
    Подробнее →
  • nil Это базовый тип данных, представляющий отсутствие полезного значения или «ничто». Используется для обозначения неинициализированных переменных и удаления ключей из таблиц.
    Подробнее →
  • true Это булево значение, обозначающее «истину». Используется в условных операторах для управления потоком программы. В Lua истинным считается любое значение, отличное от false и nil.
    Подробнее →
  • elseif Это ключевое слово для последовательной проверки нескольких альтернативных условий. Оно позволяет выполнить блок кода, если предыдущие условия оказались ложными, не закрывая конструкцию раньше времени.
    Подробнее →
  • function Это полноправный объект данных, представляющий собой блок переиспользуемого кода. Функции можно присваивать переменным, передавать как параметры и возвращать из других функций.
    Подробнее →
  • local Это ключевое слово для объявления локальных переменных с ограниченной областью видимости. Такие переменные доступны только внутри блока кода, где они созданы.
    Подробнее →
  • return Это ключевое слово, используемое для завершения выполнения функции и возврата одного или нескольких значений в место её вызова.
    Подробнее →

Примечание: в Lua 5.2 и новее также доступно ключевое слово goto для безусловных переходов.
