🔚 Ключевое слово: end

Завершение блоков кода (end)

end — это одно из самых часто встречающихся ключевых слов в языке Lua. Оно используется для закрытия (завершения) различных структурных блоков кода. Без end интерпретатор Lua не сможет понять, где заканчивается функция, условие или цикл.

📌 Важно понять: Ключевое слово end работает как «закрывающая скобка» для управляющих конструкций. Каждое if, function, for, while и do должно иметь своё парное end.

📖 Где используется end?

Конструкции, требующие завершения через end:

  • if ... then ... end — условные операторы
  • function ... end — объявление функций
  • for ... end — циклы for
  • while ... end — циклы while
  • repeat ... until — (не требует end, завершается until)
  • do ... end — явные блоки кода

💡 Примеры использования

🔹 Условный оператор if

local age = 18 if age >= 18 then print("Вы совершеннолетний") else print("Вы несовершеннолетний") end -- ← Здесь end закрывает конструкцию if-else

🔹 Объявление функции

function greet(name) return "Привет, " .. name end -- ← end закрывает тело функции -- Альтернативный синтаксис (как присваивание) local goodbye = function(name) return "До свидания, " .. name end

🔹 Цикл for (числовой)

for i = 1, 5 do print("Итерация " .. i) end -- ← end закрывает цикл for

🔹 Цикл for (обобщённый — pairs/ipairs)

local fruits = {"яблоко", "банан", "апельсин"} for index, value in ipairs(fruits) do print(index .. ": " .. value) end

🔹 Цикл while

local count = 3 while count > 0 do print("Обратный отсчёт: " .. count) count = count - 1 end -- ← end закрывает while print("Пуск!")

🔹 Явный блок do ... end

local x = 10 do local x = 20 -- локальная переменная, видима только внутри блока print(x) --> 20 end print(x) --> 10 (внешняя переменная не изменилась)

🔗 Вложенные конструкции

Очень важно правильно закрывать вложенные блоки. Каждый end соответствует ближайшему незакрытому if/function/for/while/do.

function checkNumber(n) if n > 0 then if n % 2 == 0 then return "положительное чётное" else return "положительное нечётное" end -- закрывает внутренний if elseif n < 0 then return "отрицательное" else return "ноль" end -- закрывает внешний if end -- закрывает функцию print(checkNumber(4)) --> положительное чётное print(checkNumber(-5)) --> отрицательное
⚠️ Частая ошибка: Пропуск end или несоответствие количества end количеству открытых блоков приводит к синтаксической ошибке:
'end' expected (to close 'function' at line X)

📊 Сравнение: repeat ... until vs ... end

-- Цикл repeat ... until НЕ использует end local i = 1 repeat print(i) i = i + 1 until i > 3 -- ← завершается until, а не end -- А while ... end — использует local j = 1 while j <= 3 do print(j) j = j + 1 end -- ← а здесь нужен end

🎯 Идиоматическое использование do ... end

Блок do ... end полезен для изоляции переменных и создания временной области видимости:

-- Создание сложной временной переменной без загрязнения внешней области local result do local temp = calculateComplexValue() -- дополнительная обработка temp result = temp * 2 end -- переменная temp здесь уже недоступна

⚡ Практические примеры из реального кода

🔹 Обработка данных с несколькими условиями

function getDiscount(userType, purchaseAmount) local discount = 0 if userType == "VIP" then if purchaseAmount >= 10000 then discount = 0.25 elseif purchaseAmount >= 5000 then discount = 0.15 else discount = 0.10 end elseif userType == "Regular" then if purchaseAmount >= 5000 then discount = 0.05 end end return discount end print(getDiscount("VIP", 12000)) --> 0.25

🔹 Безопасное выполнение с защитой от ошибок

function safeExecute(fn, ...) local status, result = pcall(fn, ...) if status then return result else print("Ошибка выполнения: " .. result) return nil end end -- Пример использования local riskyFunction = function() error("Что-то пошло не так") end safeExecute(riskyFunction) --> Ошибка выполнения: Что-то пошло не так

🔧 Отладка: поиск пропущенного end

💡 Совет: Если интерпретатор выдаёт ошибку 'end' expected, используйте отступы (индентацию) для визуального отслеживания вложенности. Каждый уровень вложенности должен увеличивать отступ на пробелы или табуляцию.
-- ПРАВИЛЬНО (хорошо читается) function example() if condition1 then for i = 1, 10 do -- тело end end end -- НЕПРАВИЛЬНО (легко пропустить end) function badExample() if condition1 then for i = 1, 10 do -- тело end end end
