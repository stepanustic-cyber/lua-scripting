Итерация в цикле for (in)

in — это служебное слово, используемое в обобщённом цикле for для итерации по коллекциям. Оно связывает переменные цикла с итераторной функцией (например, pairs, ipairs или пользовательским итератором).

📌 Синтаксис:
for переменная1, переменная2, ... in итератор, состояние, начальный_индекс do
    -- тело цикла
end

Итерация по таблицам

-- ipairs: итерация по последовательности (ключи 1,2,3...) local fruits = { "яблоко", "банан", "апельсин" } for i, value in ipairs(fruits) do print(i, value) end -- 1 яблоко -- 2 банан -- 3 апельсин
-- pairs: итерация по всем ключам таблицы local user = { name = "Анна", age = 25, city = "Москва" } for key, value in pairs(user) do print(key, value) end -- name Анна -- age 25 -- city Москва
-- Только ключи (игнорируем значения) for key in pairs(user) do print(key) end

Итерация по строкам

-- string.gmatch: итерация по паттернам в строке local text = "Lua - это мощный язык" for word in "(%a+)":gmatch(text) do print(word) end -- Lua -- это -- мощный -- язык
-- Разбиение строки по разделителю local csv = "apple,banana,orange" for value in "([^,]+)":gmatch(csv) do print(value) end -- apple -- banana -- orange

Создание своих итераторов

-- Итератор для диапазона чисел local function range(start, stop, step) step = step or 1 local i = start - step return function() i = i + step if i <= stop then return i end end end for i in range(1, 5) do print(i) end -- 1 2 3 4 5
-- Итератор для чтения файла построчно local function fileLines(filename) local file = io.open(filename, "r") return function() local line = file:read() if line then return line else file:close() end end end -- for line in fileLines("data.txt") do -- print(line) -- end
-- Итератор с состоянием (next) local colors = { "красный", "зелёный", "синий" } -- next — стандартная функция для итерации for key, value in next, colors, nil do print(key, value) end

Важные особенности

⚠️ Не изменяйте таблицу во время итерации:
local t = { a = 1, b = 2, c = 3 } for k, v in pairs(t) do if k == "b" then t[k] = nil -- может привести к пропуску элементов end end -- Лучше собрать ключи для удаления local toRemove = {} for k, v in pairs(t) do if k == "b" then table.insert(toRemove, k) end end for _, k in ipairs(toRemove) do t[k] = nil end
💡 ipairs vs pairs:
  • ipairs() — итерирует только по числовым ключам от 1 до N (без пропусков)
  • pairs() — итерирует по всем ключам таблицы (порядок не гарантирован)
  • ipairs() быстрее для последовательностей
